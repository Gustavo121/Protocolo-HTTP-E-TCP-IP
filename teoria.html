Request: Linha de pedido - *Método GET | POST | PUT | DELETE | PATCH | OPTIONS | HEAD | TRACE | CONNECT GET = usa para pega algum recurso, uma pagina html, dados de uma API, lista de usuarios, quando pega algum tipo de informação! POST = usa quando se vai cadastrar alguem, usa o POST para enviar ao servidor o cadastro seu ou do usuario. PUT = ja tem um usuario cadastrado, usa o PUT para editar esse usuario. DELETE = quando quiser deleta algum recurso. EX. um usuario PATCH = caso tenha que editar coisas simples, apenas dados simples de um usuario. Ex. cidade, idade. OPTIONS = usa para ver se o servidor aceita a operação que você deseja, assim ele responde se aceita ou não. HEAD = quando você faz um pedido e quer apenas os cabeçarios(meta-dados/meta-informações) TRACE = retorna para você exatamente o que você mando para o servidor, para ver se algo fico pelo caminho. Assim pode ser feito uma comparação para ver se tudo está condisendo com o que mandou. *Recurso Depois do Método você diz qual o recurso que você quer acessar Ex. /post /patch. Você diz onde você quer acessar. *Versão HTTP HTTP/0.9 HTTP/1.0 HTTP/1.1 HTTP/2 Exemplos: GET / HTTP/1.1 POST /contato HTTP/1.1 _____________________________________________________________________________________________________________________________________________ Cabeçalhos gerais - DATE | PRAGMA | CACHE-CONTOL | CONNECTION | TRAILER | TRANSFER-ENCODING | UPGRADE | VIA | WARNING DATE = data, hora , momento que foi gerada a menssagem CACHE-CONTRO = tem varios valores para controlar o cache do recurso VIA = no caminho da resposta do servidor ele preenche o header Exemplos: DATE: Sat, 13 Apr 2019 23:45:05 GMT CACHE-CONTROL: no-cache CONNECTION: keep-alive ______________________________________________________________________________________________________________________________________________ Cabeçalhos de pedido - *Preferência de resposta ACCEPT | ACCEPT-CHARSET | ACCEPT-ENCODING ACCEPT-LANGUAGE | TE ACCEPT = aqui você diz a preferencia ( html, json, image. qual formato!) o servidor atente a varios clientes diferentes, usando browser, linha de comando. assim ele se adapta a cada um.
